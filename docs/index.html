Kind Catalog

Comprehensive per-kind reference for Apple DocC's RenderNode JSON schema, built primarily from how swift-docc-render consumes and presents each kind, secondarily from swift-docc's model/doc comments. This catalog covers the content of one already-obtained RenderNode JSON file; for the archive directory that contains those files (metadata.json, index/index.json, and friends) and how to walk from one to the other, see Inside a .doccarchive. Several mechanisms cut across many kinds at once (variants, the references map, asset variant resolution, localization fallback) — see Concepts for those explained once in full.

Node kinds (top-level RenderNode.Kind) (5)

  • symbol

    Documents the top-level RenderNode kind "symbol", the DocumentationRenderNode variant used for API-reference pages (as opposed to "article"), including how swift-docc-render actually discriminates it via metadata.role/symbolKind rather than the kind field itself.

  • article

    Documents the "article" value of DocumentationRenderNode.kind — a standalone, free-form documentation page rendered by DocumentationTopic.vue, sharing the same schema and layout as symbol pages but distinguished by its metadata.role/roleHeading and hero color/icon.

  • tutorial

    Documents the top-level RenderNode.kind == "project" case — DocC's interactive tutorial page — including its Topic.vue/Tutorial.vue dispatch chain, its TutorialsPageRenderNode wire schema, and the NavigationBar dropdown's disclosure-widget-equivalent state.

  • section

    The "section" RenderNode.Kind case is never used for a full top-level RenderNode; it exclusively marks a TopicRenderReference (and its type discriminator) as pointing to an on-page @Section anchor inside a Tutorial, rendered by swift-docc-render through the fully generic ContentNode.vue -> Reference.vue -> ReferenceInternal.vue inline-link chain with no section-specific styling.

  • overview

    The `overview` RenderNode kind is the top-level page produced from a `@Tutorials` directive (a `TutorialsPageRenderNode`), rendered by swift-docc-render's `TutorialsOverview` view chain, dispatched not by inspecting the kind string in JS but by the `/tutorials/:id` route matching a bare technology-root URL.

Section kinds (RenderSectionKind) (28)

  • hero

    The hero section kind (wire value "hero", DocCArchive's IntroRenderSection) is the introductory banner rendered at the top of Tutorial and Article pages, carrying a title, optional chapter/content, and (per swift-docc's Swift model, though not the vendored JSON schema) optional background image, key image, video, project-files link, Xcode requirement, and call-to-action.

  • intro

    "intro" is a decode-only legacy alias for the "hero" RenderSectionKind — swift-docc's CodableRenderSection accepts either spelling on input but IntroRenderSection.kind is hardcoded to .hero, so swift-docc renders and swift-docc-render's dispatch tables never actually see "intro" on the wire.

  • tasks

    The `tasks` RenderSection carries the ordered list of numbered task/steps sections (`TutorialSectionsRenderSection`/`TasksSection`) that make up the body of a DocC tutorial page, each with intro content and a walkthrough of steps.

  • assessments

    The assessments RenderNode section renders a multiple-choice "Check Your Understanding" quiz for a tutorial or article, dispatched by both Tutorial.vue and Article.vue to a shared Tutorial/Assessments.vue + Quiz.vue component pair.

  • volume

    The volume section (VolumeRenderSection) groups one or more named chapters of tutorials/articles on a Tutorials Overview page, rendered by swift-docc-render's Volume.vue/VolumeName.vue/LearningPath.vue with scroll-driven "active volume" tracking synced into a shared side-nav disclosure menu.

  • contentAndMedia

    A two-column article-body layout section that pairs a block-content column with a referenced media asset (image or video), positioned leading or trailing via the mediaPosition field.

  • contentAndMediaGroup

    contentAndMediaGroup is a RenderSectionKind declared and decodable in swift-docc but never actually constructed by RenderNodeTranslator, absent from DocCArchive's vendored schema, and not consumed by any swift-docc-render component — effectively a vestigial kind.

  • callToAction

    A section-tier RenderNode kind that DocC auto-generates as a "what's next" panel at the end of a tutorial or tutorial-article page, linking to the next tutorial/article in the learning path.

  • tile

    Despite "tile" existing as a case in swift-docc's RenderSectionKind enum, it is never emitted as a discriminated top-level section; the actual object — RenderTile — is an untagged element of ResourcesRenderSection.tiles[] (kind "resources"), rendered by a Resources.vue to ResourcesTileGroup.vue to ResourcesTile.vue chain into a static grid of link cards (documentation/sampleCode/downloads/videos/forums/featured).

  • articleBody

    The articleBody render section carries the free-form prose/layout content of a tutorial-format @Article page, dispatched by swift-docc-render's Article.vue to Article/Body.vue and BodyContent.vue for layout rendering.

  • resources

    The resources section (ResourcesRenderSection) is a Tutorials Overview block, dispatched by LearningPath.vue to Resources.vue, that renders a localized "Resources" heading plus a grid of identifier-typed tiles (documentation/sampleCode/downloads/videos/forums/featured) via ResourcesTileGroup.vue and ResourcesTile.vue.

  • mentions

    The mentions RenderSection lists articles that reference a symbol, rendered by swift-docc-render as a "Mentioned in" list of up to five links resolved from raw reference-identifier strings against the render node's references map.

  • discussion

    The "discussion" RenderSectionKind is a legacy/reserved enum case in swift-docc that is declared for backward-compatible decoding but never actually emitted by current code (which always writes kind: "content" instead), and the vendored DocCArchive JSON schema doesn't even permit "discussion" as a valid kind value.

  • content

    The `content` RenderSection kind wraps an arbitrary array of RenderBlockContent — used for a symbol's Discussion/Overview/Return-Value text and an article's Overview body — and is rendered by swift-docc-render's generic ContentNode.vue block/inline dispatcher with no section-specific chrome.

  • taskGroup

    A taskGroup section is one titled cluster of curated topic links (used identically for a symbol/article page's "Topics", "See Also", and "Default Implementations" content tables), carrying an optional title/abstract/discussion plus a required list of topic-reference identifiers, with no kind discriminator on the wire.

  • relationships

    Documents the "relationships" RenderNode section kind (RelationshipsGroupSection), which lists a symbol's conformances/inheritance relationships and is rendered by swift-docc-render's Relationships.vue and RelationshipsList.vue components.

  • declarations

    Documents the "declarations" RenderNode primary-content section kind, which carries per-platform declaration token streams and optional overload ("otherDeclarations") data that swift-docc-render pulls out of the normal content flow and renders as an expandable pill list in the page hero.

  • parameters

    Documents the "parameters" RenderNode primary-content section, a static definition list of named-parameter descriptions rendered by swift-docc-render's Parameters.vue component.

  • sampleDownload

    Documents the sampleDownload RenderNode section kind, which in practice is emitted as the DocumentationRenderNode's single sampleCodeDownload property (a SampleDownload/action reference) and rendered by swift-docc-render's DocumentationTopic.vue as a call-to-action download button.

  • row

    The RenderSectionKind.row enum case named in the task is dead/unimplemented in swift-docc and swift-docc-render; the "row" wire value that actually flows through DocC output is a RenderBlockContent.row grid-layout block (produced by the @Row/@Column directives) which the page documents in full.

  • restParameters

    Documents the restParameters RenderSection, the wire-level table of REST path/query/header parameters rendered by swift-docc-render's RestParameters.vue/ParametersTable.vue components.

  • restResponses

    A symbol-page primary-content section that lists possible HTTP responses (status, reason, MIME type, declared type, and prose content) for a documented REST endpoint, rendered as a status-keyed parameter table by RestResponses.vue.

  • restBody

    Documents the restBody RenderNode section, which describes a REST endpoint's HTTP request body (MIME type, declared type, and optional multipart parameters) as rendered by RestBody.vue in swift-docc-render.

  • restEndpoint

    restEndpoint is a primary-content section that renders a REST API symbol's tokenized HTTP method + URL + path (with dynamic {parameter} segments) as a heading plus a single-line declaration-style code block, via RestEndpoint.vue delegating to the shared DeclarationSource/DeclarationToken/SyntaxToken chain.

  • properties

    The properties render section (wire kind "properties") renders a symbol's list of typed, attributed key/value entries — used today exclusively for authored dictionary-key documentation — as a two-column name/description table via swift-docc-render's PropertyTable and shared ParametersTable components.

  • plistDetails

    The plistDetails ("details") primary-content section renders a property-list/entitlement key's raw name, optional display name, and accepted value type(s) as a definition list via swift-docc-render's PropertyListKeyDetails.vue.

  • attributes

    The `attributes` RenderNode section renders a symbol's declared constraints (default value, min/max bounds, allowed values, allowed types) as a static list of label/value rows beneath an "Attributes" heading, most commonly seen on Info.plist key pages but not restricted to them.

  • possibleValues

    Documents the possibleValues RenderNode section kind, a definition-list of named property-list values (each with optional descriptive content) rendered by swift-docc-render's dedicated PossibleValues.vue component.

Block content kinds (RenderBlockContent) (17)

  • paragraph

    Documents the "paragraph" block-content RenderNode kind — a simple {type, inlineContent} wrapper rendered as a bare &lt;p&gt; element by swift-docc-render's shared ContentNode.vue dispatcher, with no dedicated component and no interactive state to replace.

  • aside

    A callout box (Note/Tip/Warning/Important/Experiment, plus render-side-only Deprecated) that wraps nested block content, dispatched by ContentNode.vue to a dedicated Aside.vue component with variant styling driven entirely by static SCSS/CSS custom properties.

  • codeListing

    Documents the codeListing block-content kind, a fenced-code-block render node whose lines, syntax language, and optional line-numbers/copy-button/wrap/highlight options are rendered by swift-docc-render's CodeListing.vue (or wrapped in a Figure when metadata.anchor is set).

  • heading

    The heading RenderBlockContent kind carries a level, plain-text title, and optional anchor slug, and is rendered by swift-docc-render's LinkableHeading.vue as a dynamic h1-h6 element with an optional in-page anchor link and hover-revealed link icon.

  • orderedList

    The orderedList RenderNode block kind carries a required items array of ListItem and an optional start integer, and is rendered by swift-docc-render's ContentNode.vue as a plain native &lt;ol start="..."&gt;/&lt;li&gt; tree with no dedicated Vue component.

  • unorderedList

    Documents the unorderedList RenderNode block kind, including swift-docc-render's task-list detection branch that swaps a plain &lt;ul&gt; for a checkbox-augmented TaskList component when any item carries a checked field.

  • step

    Documents the "step" RenderBlockContent kind, which represents a single numbered instruction inside a DocC tutorial's @Steps group and is rendered by a dedicated Step.vue/SectionSteps.vue pair with scroll-driven "active step" highlighting, rather than by the generic ContentNode.vue dispatch table.

  • endpointExample

    endpointExample is a block-content kind that renders a REST API request/response pair behind a tab switcher with per-chunk collapsible code, backed by Vue component state (currentTab, isCollapsed) in swift-docc-render's EndpointExample.vue.

  • dictionaryExample

    Documents the dictionaryExample block-content kind, a legacy render node that pairs an optional summary with a CodeExample dictionary listing, rendered in swift-docc-render via DictionaryExample.vue delegating to CollapsibleCodeListing.vue with no interactive/collapsing behavior actually wired up.

  • termList

    The termList block kind renders a sequence of term/definition pairs as a native HTML dl/dt/dd list, authored in Markdown via list items like "- term Foo: definition text".

  • table

    The table block renders a RenderNode's rows/header/alignments/extendedData fields into a static HTML table via swift-docc-render's ContentNode.vue dispatch and Table.vue wrapper, with no client-side interactivity to replace.

  • row

    The row block kind lays out a set of columns in a CSS-grid-based row, produced by an author's @Row/@Column directive pair and rendered by DocC-render's stateless Row.vue/Column.vue components with no JS-driven interactivity.

  • small

    The `small` block kind renders a paragraph of small-print (legal/license/copyright) inline content by wrapping it in a native HTML &lt;small&gt; tag with muted gray, reduced-size text.

  • tabNavigator

    The tabNavigator block RenderNode kind renders a set of titled content panels as a click-switchable tab layout, produced by an authored @TabNavigator/@Tab directive pair and consumed by swift-docc-render's TabNavigator.vue/Tabnav.vue/TabnavItem.vue component trio.

  • links

    The links block renders an authored @Links(visualStyle:) directive as either stacked declaration/abstract rows or a card grid of reference identifiers, dispatched through ContentNode.vue to LinksBlock.vue.

  • video

    Documents the RenderNode block-content kind "video", tracing its dispatch through ContentNode.vue/BlockVideo.vue/Asset.vue to the native &lt;video&gt; element rendered by VideoAsset.vue/ReplayableVideoAsset.vue, and its wire schema (Video + VideoRenderReference + RenderContentMetadata).

  • thematicBreak

    A block-tier RenderNode kind with no payload beyond its "thematicBreak" discriminator, produced by an authored Markdown horizontal rule and rendered by swift-docc-render as a bare, statically-styled &lt;hr class="thematic-break"&gt;.

Inline content kinds (RenderInlineContent) (11)

  • codeVoice

    codeVoice is the inline RenderNode kind for a short literal code span (e.g. a symbol or variable name), carrying only a `code` string that swift-docc-render renders as a bare, word-break-hinted &lt;code class="inline-code"&gt; element with no nested formatting.

  • emphasis

    The emphasis inline RenderNode kind wraps nested inline content and is rendered by swift-docc-render as a bare, unstyled &lt;em&gt; element whose italic appearance comes entirely from the browser's default user-agent stylesheet.

  • strong

    The strong inline RenderNode kind wraps nested inline content that swift-docc-render renders as a bare native &lt;strong&gt; element with no dedicated component, corresponding to double-asterisk/underscore bold Markdown text.

  • image

    The inline image kind carries a reference identifier and optional metadata (anchor/title/abstract/deviceFrame) that swift-docc-render uses to branch between a plain responsive picture element, a captioned figure, or a device-chassis frame.

  • reference

    The reference inline-content kind carries only an identifier into the RenderNode's top-level references map plus an optional title override, and DocC-Render resolves it to one of four link/span/code-styled leaf components depending on whether it's internal, active, and symbol-kind.

  • text

    The `text` inline kind is the leaf plain-text run of DocC's RenderInlineContent union, carrying only a `text` string that swift-docc-render splices directly into its parent element as a raw DOM string (or a `<br>` for a literal newline), with no wrapping element or CSS of its own.

  • newTerm

    An inline RenderNode kind meant to mark a defined term, but which swift-docc-render renders identically to plain emphasis (a bare &lt;em&gt; element) and which no authoring-side Markdown path in the pinned swift-docc checkout appears to actually construct.

  • inlineHead

    A RenderInlineContent wire kind whose schema is identical to strong/newTerm and whose renderer treats it as a plain bold &lt;strong&gt; span with no heading semantics, but which swift-docc's own Markdown compiler never appears to produce.

  • subscript

    The subscript inline RenderNode kind wraps nested inline content and is rendered by swift-docc-render as a bare native &lt;sub&gt; element with no dedicated component or CSS, though swift-docc's own Markdown compiler currently has no code path that produces it from authored text.

  • superscript

    Documents the `superscript` inline RenderNode kind, which wraps a nested inlineContent array and is rendered by swift-docc-render as a bare, unstyled-by-JS &lt;sup&gt; element via a single-line createElement dispatch in ContentNode.vue.

  • strikethrough

    The strikethrough RenderInlineContent kind wraps nested inline content in a native &lt;s&gt; element via a dedicated StrikeThrough.vue component, produced from Markdown ~~double-tilde~~ syntax, though it is oddly absent from the RenderInlineContent oneOf discriminator list in the vendored JSON Schema despite being a fully wired, real render-side and Swift-side case.

Reference kinds (RenderReference) (11)

  • image

    Documents the "image" RenderReference (ImageRenderReference) — the reference-dictionary entry holding an image's alt text and light/dark/density variants that inline "image" content nodes point to by identifier.

  • video

    Documents VideoRenderReference1, the "video" reference-tier wire object holding a video's asset variants/poster/alt text, and how swift-docc-render's Asset.vue/VideoAsset.vue/ReplayableVideoAsset.vue chain resolves it into a native, custom-controlled &lt;video&gt; element.

  • file

    The "file" RenderReference is a reference-tier record (fileName/fileType/syntax/content/highlights) that a tutorial step's CodePreview.vue looks up by identifier from the references dictionary and spreads as props into the same CodeListing.vue component used for plain codeListing blocks.

  • fileType

    FileTypeRenderReference is a references-dictionary entry (displayName + Base64 icon for a class of file) that is schema-defined and decodable in swift-docc but never constructed by the archive generator and never consumed anywhere in swift-docc-render's Vue components.

  • xcodeRequirement

    Documents the xcodeRequirement RenderReference, a title/url pointer to a required Xcode download shown as a metadata chip in a tutorial's hero section.

  • topic

    The "topic" reference kind is the wire shape (TopicRenderReference) used in a RenderNode's references dictionary to represent a link to another documentation page, resolved by identifier and rendered by swift-docc-render's TopicsTable/TopicsLinkBlock components into a titled link row with optional abstract, badges, and role icon.

  • section

    The "section" RenderReferenceType is TopicRenderReference's discriminator value for a reference that points at an in-page heading anchor rather than a whole page, and swift-docc-render renders it identically to an ordinary topic-link reference with no dedicated component.

  • download

    The `download` RenderReference kind is a references-map entry (URL + optional SHA-512 checksum) that a tutorial's "Project files" link or a symbol/article's sample-code download button resolves to, produced from a `@Tutorial(projectFiles:)` directive argument or a page's sample-code project asset.

  • link

    LinkRenderReference is the reference-tier object DocC emits for authored Markdown links to external (non-doc:) URLs, resolved through the same generic inline `reference` dispatch and Reference.vue component as internal topic links, but always rendering as a plain external anchor.

  • externalLocation

    externalLocation is an alternate wire value for the same DownloadRenderReference schema/struct used by "download", denoting a downloadable resource hosted off-site, and it is consumed by swift-docc-render generically as a URL/title pair with no type-specific branching.

  • unresolvable

    The unresolvable RenderReference type represents a link DocC could not resolve at build time (e.g. a symbol from a module lacking symbol-graph data), carrying only a title and identifier and rendering in swift-docc-render as unlinked plain text.
