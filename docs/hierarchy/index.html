Page Hierarchy Guide

A companion to the kind catalog: same underlying RenderNode schema data, but organized top-down by page (route → component chrome → schema content → leaf inline nodes) instead of flat by kind.

Top-level pages

Top-level pages (6)

  • Documentation Topic

    The symbol/article route (/documentation*), dispatched by views/DocumentationTopic.vue to the shared DocumentationLayout/Topic chrome, covering both kind: "symbol" and kind: "article" RenderNodes.

  • Tutorial (Project)

    The interactive DocC tutorial page (wire kind: "project"), dispatched by views/Topic.vue to components/Tutorial.vue and its hero/tasks/assessments/callToAction section chain.

  • Tutorial Article

    The free-standing tutorial-catalog article page (wire kind: "article", tutorial variant), dispatched by views/Topic.vue to components/Article.vue — a distinct rendering path from the Documentation Topic's own kind: "article".

  • Tutorials Overview (Table of Contents)

    The technology-root "table of contents" page (wire kind: "overview"), matched purely by URL shape (/tutorials/:id) to views/TutorialsOverview.vue, listing a technology's volumes/chapters and resources.

  • Not Found (404)

    The ordinarily-reachable catch-all route rendered by views/NotFound.vue, with no RenderNode-schema content at all — just a single static, localized "page not found" string.

  • Server Error (500)

    The purposefully-unreachable-by-URL fallback route rendered by views/ServerError.vue, reachable only via explicit named navigation, showing a single static "unknown error" string with no schema content.
