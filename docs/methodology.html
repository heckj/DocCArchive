How to check for staleness

Run these commands and compare to the pinned SHAs above:

git -C ~/src/swift-project/swift-docc log -1 --format=%H
git -C ~/src/swift-project/swift-docc-render log -1 --format=%H

If either differs, scope the review to what actually changed rather than re-checking everything:

git -C ~/src/swift-project/swift-docc diff 8b8d8268aa13bd1cfcd081c5d0644437fe4880e7..HEAD --stat -- Sources/SwiftDocC/Model/Rendering/
git -C ~/src/swift-project/swift-docc-render diff 4a9efad63734d40cf742da9f36c99b9bc57305f8..HEAD --stat -- src/components/ src/views/

Any touched files that a kind page cites are the ones needing re-review; re-run the corresponding per-kind generation step for just those kinds.

Known gaps

DocCArchive's vendored schema commit predates the pinned swift-docc checkout above — flagged per-page where relevant. 34 of 72 kind pages use a hand-synthesized (non-fixture) example JSON because no real fixture exercising that kind was found in DocCArchive's or swift-docc's test suites at generation time; these are labeled inline on their pages.

This site's scope is the RenderNode schema for one already-obtained JSON file; it does not cover the .doccarchive directory that file lives in. See Inside a .doccarchive for metadata.json, index/index.json, and the other archive-level files.

Four mechanisms (variants/variantOverrides, the references map, asset variant resolution, localization fallback) cut across many kind pages at once and were previously explained in full on only one page each. See Concepts for the consolidated versions; affected kind pages carry a .see-concept forward link back to the relevant one.

docs/manifest.json is hand-maintained, not generated — there is no script that keeps it in sync with the 72 kind pages. Whenever a kind page's tier, verdict, or synthesized-flag changes, update the corresponding entry in manifest.json in the same change. The "30 vs 34" synthesized-count drift above is exactly the kind of bug this file is prone to; periodically re-running the grep-based consistency checks (per-page tier-badge/wire-value/appears-on/verdict/ synthesized-note markers vs. the manifest entries) is the recommended way to catch drift before it accumulates.
