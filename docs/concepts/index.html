Cross-cutting concepts

The Kind Catalog documents each RenderNode schema kind in isolation, but four mechanisms cut across many kinds at once and are easy to miss if you only ever read one page at a time. Each page below explains one mechanism once, in full, with a worked example spanning more than one kind; the kind pages that touch it link back here rather than re-explaining it.

  • variants & variantOverrides

    How one RenderNode serves both a Swift and an Objective-C presentation — a routing table for per-language paths plus a JSON-Patch mechanism for in-place content overrides.

  • The references map

    Why almost every link, image, and download node on the wire carries only an opaque identifier, and how that identifier resolves against one shared top-level dictionary.

  • Asset variant resolution

    How a flat array of {url, traits} entries becomes light/dark, 1x/2x/3x <picture>/srcset markup for images, video sources, and hero backgrounds.

  • Localization fallback ($t())

    How DocC-generated chrome strings (aside labels, control button names) are looked up from locale files at render time, with English as the runtime fallback, rather than ever appearing on the wire.
